## Design Context

### Users
个人加密货币交易者，每日使用 LiquidityOS 进行市场状态判断和交易决策。高频使用、长时间盯盘、依赖扫描效率，通过 L0→L4 五层决策漏斗从宏观到微观逐层聚焦。桌面端专用（1024px+），无移动端需求。

### Brand Personality
**冷静 · 清晰 · 可信**
像 Linear/Stripe Dashboard 那样——现代但不花哨，重信息层级，零装饰噪音。界面服务于快速判断和快速操作，不服务于视觉展示。

### Emotional Goals
- **掌控感**：所有信息都在掌握中，决策有据可依
- **安全感**：系统可靠、数据可信、风险可控
- **专注感**：噪音被过滤，只看到当下最重要的信号

### Aesthetic Direction
- **视觉基调**：Industrial-utility，信息密度优先，留白服务层级而非装饰
- **参考**：Linear (clarity), Stripe Dashboard (trust), Bloomberg (density when needed)
- **反参考**：
  - 不要像 DeFi 营销页（紫蓝渐变、霓虹发光、玻璃态、大屏装饰）
  - 不要像传统券商 App（拥挤老式表格、低对比度、过时 UI）
  - 不要像 AI 生成的 Dashboard（hero metric 模板、同质化卡片网格、gradient text）
- **主题**：暗色/亮色双模式同等重要，必须同质量

### Design Principles
1. **信号优先**：颜色只承担语义职责（bull/bear/neutral），不做无意义装饰
2. **1-2 秒可识别**：信息层级必须在扫描速度内建立，重要数据稳定且明显
3. **克制一致**：字体、数字、状态标签有严格规则，不堆砌一次性样式
4. **功能效率 > 美观**：交易系统首先服务"快速判断"，不为"高级感"牺牲密度
5. **健壮可信**：考虑异常状态、边界情况、弱网，让系统看起来可靠

### Key Design Decisions (User-confirmed)

1. **Dashboard Hero 保留** — Hero 是所有层信号的总结入口，有存在价值。但需要从"AI metric 展台"重构为"决策简报"。
2. **SummaryCard 不同构** — L0/L1/L2/L3 是四种完全不同的数据结构，应根据各自数据特征使用不同视觉表达，不要强套同一个卡片模板。
3. **L4 工作台是最高频使用模块** — L4 与 L1/L2/L3 是并列模块，打开即到执行工作台（MemeRadar / WatchStation / AlphaScanner）。
   - 原 L4 顶部的 Hero Signal + BTC Anchor + Market Context（L1/L2/L3/FG）→ 移到 Dashboard 底部，默认折叠
   - 原 L4 底部的 Review & Diagnostics → 重命名为"Alpha 决策历史"，保留在 L4 底部
   - 这是当前系统中问题最多、最需要优先重构的区域。
